Introduction

Why Study JavaScript?

JavaScript is one of the 3 languages all web developer must learn:

  1. HTML to define the content of web pages
  2. CSS to specify the layout of web pages
  3. JavaScript to program the behavior of web pages

JavaScipt Can Change HTML Content

One of many JavaScript HTML methods is getElementById().
The example below "finds" an HTML element (with id="selector"), and changes the element content (innerHTML) to "Hello JavaScript":

document.getElementById("selector").innerHTML = "Hello JavaScript";

JavaScript Can Change HTML Styles (CSS)

Changing the style of an HTML element is a variant of changing an HTML attribute.
The example below "finds" and HTML element (with id="selector"), and changes the font size to 35px:

document.getElementById("selector").style.fontSize = "35px";
Where to

The <script> Tag

In HTML, JavaScript code is inserted between <script> and </script> tags.

<script>
 document.getElementById("selector").innerHTML = "My First JavaScript";
</script>

Where to place JavaScript code

JavaScript code can be placed in:

  • In the <head> section of an HTML document
  • In the <body> section of an HTML document
  • In external files with the .js extension

External JavaScript Advantages

Placing scripts in external files has some advantages:

  • It separates HTML and code
  • It makes HTML and JavaScript easier to read and maintain
  • Cached JavaScript files can speed up page loads

To add several script files to one HTML documetn - use several <script> tags:

<script src="myScript1.js"></script>
<script src="myScript2.js"></script>

Syntax

JavaScript Values

The JavaScript syntax defines two types of values:

  • Fixed values, which called Literals
  • Variable values, which called Variables

JavaScript Literals

The two most important syntax rules for fixed values are:

  1. Numbers are written with or without decimals: 10.49
    1990
  2. Srings are text, written within double or single quotes: "Chris Bumstead"
    'Chris Bumstead'

JavaScript Variables

In programming language, variables are used to strore data values.
JavaScript uses the keywords var, let and const to declare variables.
An equal sign is used to assign values to variables.
In this example, x is defined as a variables. Then, x is assigned (given) the value 6:

let x;
x = 6;

JavaScript Operators

JavaScript uses arithmetic operators ( + - * / ) to compute values:

(17 + 6 / 2) * 10

JavaScript uses an assignment operator ( = ) to assign values to variables:

let x, y, z;
x = 5;
y = 11;
z = 9;

JavaScript Expressions

An expression is a combination of values, variables, and operators, which computes to a value.
The computation is called evalution.
For example, 10 * 6 evaluates to 60, x / 5 evaluates to 8 (x is assigned the value of 40), "Chris" + " " + "Bumstead" evaluates to "Chris Bumstead":

10 * 6;
let x;
x = 40;
x / 5;
"Chris + " " + "Bumstead";

JavaScript Keywords

JavaSCript keywords are used to identify actions to be performs.
The let keyword tells the browser to creat a variables:

let x, y;
x = 10 + 2;
y = x / 3;

JavaScript Identifiers / Names

Indentifiers are JavaScript names, are used to name variables, keywords, and functions.
The rules for legal names are the same in most programming languages. A JavaScript name must begin with:

  • A letter (A-Z or a-z)
  • A dollar sign ($)
  • An underscore (_)

Subsequent characters may be letters, digits, underscore, or dollar signs.
Numbers are not allowed as the first character in names. This way JavaScript can easily distinguish indentifiers from numbers.
Hyphens (-) are not allowed in JavaScript names. They are reserved for subtractions.

JavaScript is Case Sensitive

All JavaScipt indentifiers are case sensitive.
The variables lastName, lastname and LastName, are three different variables.
JavaScript does not interpret LET or Let as the keyword let.
Moreover, JavaScript programmers tend to use Camel Case (that starts with a lowercase letter) to join multiple words into one variable name:

let studentName, masterCard, isDigit;
Comment

Are all JavaScript statements executed?

Not all JavaScript statements are executed.
Code after double slashes // or between /* and */ is treated as comment.
Comments are ignored, and will not be executed.
JavaScript comments can be used to explain JavaScript code, and make it more readable.
JavaScript comments can also be used to prevent execution, when testing alternative code.

Single Line Comments

Single line comments start with //.
Any text between // and the end of the line will be ignore by JavaScript (will not be executed).
This example uses a single-line comment before the first code line and another at the end of the second code line to explain the code:

// Change heading:
document.getElementById("myHeading").innerHTML = "My First Heading";

let x = 10; // Declare x, give it the value of 5;

Multi-line Comments

Multi-line comments start with /* and end with */.
Any text between /* and */ will be ignored by JavaScript.
This example uses multi-line comment (a comment block) to explain the code:

/*
The code below will change
the heading with id = "myHeading"
and the paragraph with id = "myParagraph"
in my web page:
*/
document.getElementById("myHeading").innerHTML = "My First Heading";
document.getElementById("myParagraph").innerHTML = "My first paragraph.";

Using Comments to Prevent Execution

Using comments to prevent execution of code is suitable for code testing.
Adding // in front of a code line changes the code lines from an executable line to a comment.
This example uses */ to prevent execution of one of the code lines:

// document.getElementById("myHeading").innerHTML = "My First Heading";
document.getElementById("myParagraph").innerHTML = "My first paragraph.";

This example uses a comment block (/* and */) to prevent execution of mutiple code lines:

/*
document.getElementById("myHeading").innerHTML = "My First Heading";
document.getElementById("myParagraph").innerHTML = "My first paragraph.";
/*
